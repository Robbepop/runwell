pub fn match_select_multi<T>(
    self,
    selector_type: IntType,
    selector: Value,
    result_types: T
) -> Result<MatchSelectInstructionBuilder<'a, 'b>, Error> where
    T: IntoIterator<Item = Type>, 

Selects from the given target result values or the default given a selector.

Note

This can be used in order to construct MatchSelectInstr instructions where match arms are required to return more than just a single value. Since the construction is more complex than with just a single returned value per match arm users are free to use the simpler match_select or bool_select constructors.

Note

This mirrors the branching table construct but using value semantics instead of taking branches.
